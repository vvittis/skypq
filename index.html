📦 Incremental Package Queries

About Demo Publications People Code
Incremental Package Queries extends database systems with efficient constrained optimization over dynamic data. We develop methods to maintain optimal solutions incrementally—avoiding expensive recomputation when data or query parameters change.

A package query returns a collection of tuples that collectively optimize an objective function while satisfying constraints. Package queries enable in-database prescriptive analytics across domains like finance, healthcare, logistics, and cloud computing.

📦 Continuous VM Provisioning

Jordan is a platform engineer provisioning VMs for a data pipeline requiring at least 800 Compute Units. Jordan wants the cheapest bundle meeting this requirement, with practical constraints: 8–12 instances for fault tolerance and quota limits, and at most 2 copies of any single VM type.

The cloud provider's catalog contains 6,500 VM offers—yet Jordan's package will contain only 8–12 of them. Spot prices fluctuate continuously, requiring frequent re-optimization. At any moment, most options are obviously poor deals. Our methods identify which options actually matter, reducing the catalog by over 99% while guaranteeing the same optimal solution.

Package Query Language (PaQL)

SELECT PACKAGE(*) AS P FROM vm_offers REPEAT EACH P.offer_id AT MOST 2 SUCH THAT SUM(P.compute_units) >= 800 AND COUNT(P.*) BETWEEN 8 AND 12 MINIMIZE SUM(P.cost_per_hour)
